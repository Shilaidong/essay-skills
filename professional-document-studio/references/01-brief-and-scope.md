# 任务合同、范围与输入检查

## 1. 先把“写文档”改写成可验收任务

开始前形成一页内部任务合同：

| 字段 | 要回答的问题 |
|---|---|
| 文档类型 | 报告、方案、计划、纪要、白皮书、案例分析，还是其他？ |
| 目标读者 | 谁会阅读、批准、执行或转发？ |
| 核心目的 | 读者读完后要理解、决定或行动什么？ |
| 使用场景 | 汇报、申请、内部决策、对外发布、存档还是投标？ |
| 输出格式 | DOCX、PDF、Markdown、PPT，或组合？ |
| 语言与语气 | 中文/英文；正式、研究型、简洁、说服型？ |
| 篇幅 | 页数、字数或阅读时间？ |
| 必含模块 | 用户指定的章节、表格、图表、案例、时间轴等 |
| 禁用内容 | 不写哪些对象、案例、方案或敏感信息？ |
| 研究范围 | 是否联网；允许哪些来源；截止到何时？ |
| 视觉要求 | 字体、配色、封面、页眉页脚、图表风格 |
| 交付附件 | 来源台账、数据、图片、可再生成 YAML、日志？ |
| 截止时间 | 何时需要；是否分阶段交付？ |

## 2. 什么时候必须提问

只在以下信息缺失且会显著改变结果时提问：

- 目标读者完全未知；
- 需要外部研究但用户明确禁止联网或来源范围不清；
- 输出格式未说明，且用户明显需要文件；
- 预算、法律责任、身份或敏感数据会改变建议；
- 多个方向互相冲突，无法在一个文档中同时成立；
- 用户要求修改某个文件，但文件未提供或无法读取。

优先只问一个聚合问题，而不是连续追问。

## 3. 什么时候可以直接继续

可以采用显式假设继续：

- 没有指定页数：按内容复杂度设置合理篇幅；
- 没有指定字体：中文 DOCX 默认宋体；
- 没有指定引用格式：正文简洁引用 + 末尾来源表；
- 没有指定颜色：采用克制的单一强调色；
- 没有指定图表：只在确实提高理解时加入；
- 没有指定目录：使用决策优先的默认结构。

在文档说明或交付备注中列出影响较大的假设。

## 4. 输入文件检查清单

对每个文件记录：

- 文件名与路径；
- 格式；
- 页数、行数或工作表数量；
- 创建日期和版本；
- 是否可解析；
- 是否包含表格、图表、扫描页或图片；
- 是否含个人信息、机密信息或版权内容；
- 是否与其他材料冲突；
- 是否需要 OCR；
- 是否是最终版本。

## 5. 输入不足时的最小可行输出

若资料不足，仍可交付：

1. 已核验事实；
2. 暂定结构；
3. 关键缺口；
4. 需要补充的材料；
5. 可先执行的下一步；
6. 明确标注的假设。

不要用空洞文字填充缺失内容。
