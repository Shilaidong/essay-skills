# 分析、决策与风险框架

## 1. 不从“写得长”开始

先回答三个问题：

1. 文档要支持什么决定？
2. 哪些事实最可能改变这个决定？
3. 读者采取行动前还需要知道什么？

## 2. 常用分析骨架

### 2.1 现状—目标—差距—路径

适合规划、提升、转型和整改类文档。

```text
当前状态
→ 目标状态
→ 差距原因
→ 优先动作
→ 资源与时间
→ 风险与监测
```

### 2.2 选项—标准—权衡—推荐

适合选校、选供应商、选方案、投资或产品决策。

步骤：

1. 定义选项；
2. 定义评价标准；
3. 说明标准权重或优先级；
4. 对比证据；
5. 识别不可兼得的取舍；
6. 推荐并给出触发转向的条件。

### 2.3 问题—原因—机制—影响—对策

适合诊断、复盘和风险报告。

不要把症状当原因，也不要把“缺乏重视”当作最终根因。

### 2.4 输入—活动—产出—结果—影响

适合项目设计和影响评估。

- 输入：资源；
- 活动：做什么；
- 产出：直接交付；
- 结果：行为或制度变化；
- 影响：更长期变化。

浏览量、活动次数等通常是产出，不等于影响。

## 3. 证据链模板

每项重要建议至少包含：

```text
结论：建议做什么
证据：哪些事实支持
解释：为什么这些事实重要
取舍：放弃或承担什么
行动：谁在何时做什么
指标：如何知道有效
风险：什么会使建议失效
```

## 4. 多方案写法

仅在用户需要多个方案时使用。

每个方案必须独立包含：

- 适用条件；
- 目标；
- 关键动作；
- 资源；
- 里程碑；
- 风险；
- 退出或转向条件。

之后再给：

- 共享底座；
- 主要差异；
- 决策矩阵；
- 最晚选择时间；
- 选择标准。

不得为了“看起来全面”强行制造多个方向。

## 5. 风险登记

每项高风险建议记录：

| 风险 | 概率 | 影响 | 预警指标 | 缓解措施 | 备用方案 | 负责人 |
|---|---|---|---|---|---|---|

风险类型：

- 事实/信息风险；
- 时间风险；
- 资源风险；
- 执行风险；
- 合规与隐私风险；
- 依赖方风险；
- 声誉风险；
- 质量风险。

## 6. 量化原则

数字应说明：

- 基线；
- 目标；
- 时间范围；
- 计算方式；
- 数据来源；
- 是否为目标或已完成结果。

不要使用看似精确但没有依据的数字。

## 7. 不确定性表达

可用：

- “现有证据支持……”
- “更可能的解释是……”
- “在……前提下，建议……”
- “当前无法验证……”
- “该结论对……较敏感。”

避免：

- “一定”“绝对”“无敌”“保证”；
- 在证据不足时给出精确概率；
- 把缺失信息隐藏在流畅文字中。

## 8. 推荐优先级

可按以下标准排序：

1. 对核心目标的影响；
2. 时间敏感性；
3. 可逆性；
4. 资源需求；
5. 依赖条件；
6. 风险；
7. 可验证性。

执行摘要中通常只保留 3–5 项最高优先级动作。
