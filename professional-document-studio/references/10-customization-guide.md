# 调用时加入领域模块

本 Skill 不预设行业、主题或固定章节。用户在每次调用时提供差异化要求。

## 1. 最小调用信息

```text
主题：
文档类型：
目标读者：
核心目的：
输入材料：
输出格式：
```

## 3. 多方向或多方案

只有用户明确要求时加入：

```text
请分别写方案A与方案B。每个方案都要独立包含目标、方法、活动、资源、里程碑、风险和退出条件；最后给比较矩阵与选择时间点。
```

## 4. 案例模块

只有用户明确要求时加入：

```text
加入两个公开可核验案例。可匿名化和重新组织，但必须保留来源台账，不得冒充自有案例；正文聚焦可迁移方法和限制。
```

## 5. 高度定制的 Word

```text
最终生成中文 Word：A4、正文宋体10.5号、标题宋体加粗、单一深色强调色；大章节另起一页；表格首行重复；甘特图以高清PNG插入；完成后逐页渲染检查。
```

## 6. 只修改已有文档

```text
请保留原有事实和章节编号，重构逻辑、删除重复、统一宋体、优化表格并加入执行摘要。另存新文件，不覆盖原件；渲染检查后交付。
```

## 7. 不联网模式

```text
不要联网。只使用我提供的材料。无法验证的事实标为“用户提供”或“待核验”，不要补写猜测。
```

## 8. 研究优先模式

```text
先完成来源台账和一页研究结论，再开始写 Word。所有易变事实只使用当前官方来源。
```
