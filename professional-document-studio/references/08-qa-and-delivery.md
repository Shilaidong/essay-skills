# 质量检查、隐私与交付

## 1. QA 分四层

### 1.1 内容 QA

- 是否回答用户原始问题；
- 是否遗漏必含模块；
- 结论是否前置；
- 建议是否可执行；
- 限制和风险是否说明；
- 多个方案是否各自完整；
- 是否存在自相矛盾。

### 1.2 事实 QA

- 名称、数字、日期和单位一致；
- 易变事实已核验；
- 引用支持对应主张；
- 表格、图表和正文一致；
- 目标与已完成结果分开；
- 推断有足够证据且明确标注。

### 1.3 文件结构 QA

- 标题样式正确；
- 目录层级正确；
- 图片路径有效；
- 表格头重复；
- 页码、页眉页脚正确；
- 无残留批注、修订或占位符；
- 无损坏链接；
- 文档可重新打开。

### 1.4 视觉 QA

渲染为逐页 PNG 后检查：

- 每页完整；
- 无裁切或重叠；
- 无空白页；
- 无缺字；
- 表格可读；
- 图片清晰；
- 标题不孤立；
- 页码连续；
- 甘特图可读；
- 章节分隔合理。

## 2. 建议 QA 命令

```bash
python scripts/qa_docx.py report.docx \
  --render-dir _rendered \
  --json-output qa-report.json \
  --strict
```

查看：

- `_rendered/contact-sheet.png`；
- 每张 `page-*.png`；
- `qa-report.json`。

修改后重新生成和渲染。

## 3. 占位符扫描

交付前搜索：

```text
TODO
TBD
待补充
信息待补齐
XX
[插入]
[[REF:
turn0search
sandbox:
```

并判断哪些应删除，哪些应转成明确的“待确认事项”。

## 4. 隐私与元数据

检查：

- 作者、公司、最后修改人；
- 评论和修订；
- 隐藏文本；
- 文件路径；
- 图片 EXIF；
- 个人电话、邮箱、地址；
- 未成年人信息；
- 内部项目名；
- 原始数据嵌入；
- 链接权限。

需要时生成去标识化版本，不覆盖完整内部版。

## 5. 可访问性

至少检查：

- 标题层级；
- 表格表头；
- 图片替代文字；
- 链接文字；
- 颜色对比；
- 阅读顺序；
- 字号。

## 6. 交付文件命名

推荐：

```text
项目名_文档类型_YYYYMMDD.docx
项目名_时间轴_YYYYMMDD.png
项目名_来源台账_YYYYMMDD.csv
```

最终交付中不要使用临时名称、随机哈希或“final2”。

## 7. 交付包

按需包含：

- 最终 DOCX；
- PDF（仅用户要求）；
- 图表 PNG；
- 来源台账 CSV；
- 数据或代码；
- report.yaml；
- README 或使用说明。

内部渲染页、缓存和日志默认不交付。

## 8. 最终回复

最终回复应：

- 直接给下载链接；
- 简要说明主要更新或文件内容；
- 若有未解决限制，明确指出；
- 不复述整份文档；
- 不声称“绝对无误”。
