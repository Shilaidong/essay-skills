# Report Blueprint

Use this as the default structure unless the user supplies a stricter format.

## Cover

- Student name.
- Target entry year.
- Target country/degree.
- Date.
- Prepared for.
- Confidentiality note if appropriate.

## 1. Executive summary

Include:

- One-sentence positioning.
- Recommended route.
- Primary risks.
- Immediate next 30 days.
- Decision gates.

## 2. Student information and missing items

Table:

```text
Item | Current information | Status | Implication | Follow-up
```

## 3. Admissions-reader diagnosis

Subsections:

- Current read.
- Target read.
- Strongest assets.
- Weak points.
- Missing proof.
- Three to five risks.

## 4. Route architecture

For single-track:

- Core direction.
- Evidence needed.
- Why it fits.

For dual/multi-track:

- Shared evidence core.
- Track A plan.
- Track B plan.
- Decision gate.
- Pivot rules.

## 5. School/program strategy

Table columns:

```text
Tier | Institution | Location | College/School | Program | Fit logic | Requirements | Round | Risks/notes
```

If safeties are excluded, state that the plan omits safeties by user request.

## 6. Academic plan

Include:

- Course strategy.
- GPA/rank target.
- Exam plan.
- Weak-subject repair.
- Academic deliverables.

## 7. Testing and language plan

Include:

- Required vs recommended tests.
- Target ranges.
- Preparation timeline.
- Test windows.
- Stop rules.

## 8. Background-development plan

For each main line:

- Name.
- Purpose.
- Problem.
- Method/tools.
- Milestones.
- Deliverables.
- Metrics.
- Verification.
- Risks and fallback.
- Application translation.

## 9. Essays and recommendations

Include:

- Main essay theme.
- School-specific supplements.
- Resume/activity list logic.
- Recommendation choices.
- Recommender evidence packet.

## 10. Timeline

Use:

- Month-by-month plan.
- Weekly plan for intense periods.
- Milestones.
- Gantt image.

## 11. Optional case/benchmark module

Each case:

- Anonymous label.
- Route/program relevance.
- Verified public pattern.
- What can be borrowed structurally.
- What should not be copied.
- Implication for this student.

## 12. Risk register

Table:

```text
Risk | Trigger | Severity | Mitigation | Owner | Deadline
```

## 13. Verification checklist

List facts to verify before submission.

## 14. Sources appendix

Client-facing source list or ledger summary.

## Writing style

- Decision-first.
- Specific.
- Evidence-based.
- Clear distinction between current facts and target outcomes.
- No admissions guarantee language.
- No filler phrases like “enhance competitiveness” without explaining how.
