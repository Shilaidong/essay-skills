# Case and Benchmark Module Policy

## Purpose

Cases are used to show strategic patterns, not to create myths or guarantee outcomes.

A case module can help the user understand:

- What kind of evidence strong applicants tend to show.
- How activities become narrative proof.
- How depth differs from cosmetic scale.
- What this student should emulate structurally.
- What this student should not copy.

## Ethical boundary

Allowed:

- Use public cases as anonymous benchmarks.
- Summarize and synthesize public information.
- Remove unnecessary identifying details.
- Preserve source provenance internally.
- Say “benchmark case” or “anonymous public benchmark.”

Not allowed:

- Present public internet cases as proprietary in-house cases.
- Invent details to make a case stronger.
- Claim the activity caused admission.
- Copy text from a source at length.
- Use private personal data without permission.

## Case template

```text
Case label:
Relevant route/program:
Verified public pattern:
Evidence shown:
Strategic lesson:
What not to copy:
How to adapt for this student:
Source notes: kept in internal source ledger / included in appendix if requested
```

## Adaptation rule

Do not copy the surface form. Extract the underlying structure.

Example:

- Surface: “founded a city blog.”
- Structure: “used a personal observation to build a long-term public-facing research and storytelling platform with identifiable outputs.”

The student's plan should be built from their own background, evidence, and constraints.
