# Core Activity and Competition Pathways Guide

## Purpose

This reference helps an agent design applicant-specific activity and competition plans. It replaces one-size-fits-all competition planning. ISEF, Olympiads, Conrad, robotics, debate, writing contests, service projects, sports leadership, and other activities should appear only when they fit the student’s evidence, major direction, timeline, and risk tolerance.

## General principle

Activities and competitions are evidence channels. They should produce one or more of the following:

- academic depth
- intellectual vitality
- measurable impact
- technical/product output
- third-party validation
- leadership progression
- recommendation-letter material
- essay-worthy growth or identity evidence

Never treat a competition as a guaranteed result. Always explain the advancement mechanism or state that no formal advancement exists.

## Pathway architecture

Use this chain for every designed pathway:

```text
preparation/output -> submission/selection -> possible advancement -> final/recognition -> application evidence -> fallback use
```

If advancement exists, name the exact type:

- score/cutoff advancement
- regional -> national -> international rounds
- affiliated fair or official representative selection
- nomination/recommendation pool
- application review plus interview
- team qualifier
- judges' award leading to final round
- editorial/publication review
- showcase/demo day selection

If there is no advancement:

- write `无正式晋级机制`
- explain value through user numbers, partners, media, publication, portfolio, code, prototype, report, event scale, coach/mentor recommendation, or school/community adoption

## Field-specific patterns

### STEM research / science fair

Good for students with a technical question, data, experiments, code, or lab evidence.

Path:

1. Define research question and variables.
2. Collect data or build prototype.
3. Produce paper, poster, abstract, data book, code, demo.
4. Submit to school/local/city/province fair or recognized selection if available.
5. Advance only through official finalist/representative selection and compliance review.
6. Use project results in application even if no major award.

Required compliance when relevant:

- research plan
- student contribution statement
- lab/data notebook
- risk assessment
- human-subject/minor/data-privacy approval if applicable
- battery/electrical/chemical/biological safety if applicable
- AI use disclosure if applicable
- display/safety rules for in-person fairs

ISEF-specific details should be added only if the student’s target and pathway genuinely involve ISEF or an affiliated fair.

### Engineering design / innovation / entrepreneurship

Good for students with a prototype, design problem, user pain point, or market/social application.

Path:

1. Prototype or system concept.
2. User testing/customer discovery.
3. Technical validation and failure iteration.
4. Pitch deck, innovation canvas, website/video, demo.
5. Preliminary submission -> regional/national/global final if applicable.
6. Fallback: publish demo, GitHub, user report, school showcase, or partner pilot.

Risks:

- team role unclear
- overclaiming business traction
- product too conceptual
- prototype not functional

### Subject competitions

Good for students needing hard academic validation.

Path:

1. Diagnostic baseline.
2. Syllabus map and weekly training plan.
3. First-round exam.
4. Award/cutoff.
5. Next round if the contest has a formal progression.
6. Use results to support major readiness.

Include:

- current baseline
- target award band
- training hours
- score threshold if official or historically known
- whether advancement exists
- fallback if no award

### Robotics / hackathon / maker competitions

Good for students with coding, engineering, design, and team execution.

Path:

1. Team and role definition.
2. Build season or hackathon sprint.
3. Engineering notebook / GitHub / demo.
4. Regional qualifier/showcase.
5. Award/final if applicable.

Risks:

- student’s individual contribution hidden by team
- too much time cost
- school team dependency

### Humanities / writing / debate / MUN / speech

Good for students with writing, argumentation, policy, language, history, philosophy, or public speaking evidence.

Path:

1. Topic portfolio.
2. Draft/brief/case development.
3. Coach/judge/editorial feedback.
4. Submission/tournament.
5. Regional/national rounds or publication if applicable.

Evidence:

- writing samples
- awards
- publication
- judge feedback
- speaking recordings
- leadership in team preparation

### Social impact / community service

Good for students with a real community problem and sustained operations.

Usually no formal advancement. Value comes from measurable impact.

Path:

1. Identify target group and pain point.
2. Pilot solution.
3. Build operating system.
4. Measure reach, retention, learning gains, funding, partners, or adoption.
5. Publish impact report.
6. Use partner letters and metrics in applications.

Risks:

- inflated numbers
- one-off volunteering
- lack of sustainability
- minors/data/privacy issues

### Sports / arts / performance

Good for students with sustained commitment, high skill, team role, or performance record.

Path:

1. Document historical progression.
2. Define current role and next leadership step.
3. Build a system: training, rehearsal, recruitment, match/report review, event planning.
4. Enter league/showcase/competition if relevant.
5. Collect coach/teacher feedback and measurable contribution.

Value often comes from discipline, growth, leadership, and team impact rather than formal awards.

## Report-writing rules

- Name only 3–5 core pathways. Do not list random activities just to look busy.
- Prioritize depth, continuity, and evidence.
- For each pathway, define the student’s personal role.
- If there is an advancement route, write it as conditional.
- If there is no advancement route, explain output and evidence value.
- Always include a fallback path so the plan still works if the student does not advance.
