# 04 · 检查角度

本文件用于 HTML 完成后的审查。按角度检查，而不是只问“有没有做完”。任何失败项都要修正后再交付。

## 1. 事实保真

- 没有发明数字、日期、来源、作者、公司、案例、排名、地图点位、图表数据或结论强度。
- 视觉隐喻没有变成事实暗示。
- 每个关键判断都能回到原报告。
- 条件、限制、hedge 没有被删成绝对断言。
- 缺失信息被标为“原文未给出”“数据缺口”“需补充”，没有被补写。

## 2. 主题契合

- 首屏以主题领域中有代表性的事物或机制开篇。
- 这套风格换到完全不同主题上会不成立。
- 配色、字体、纹理、动效和不可遗忘点都能追到领域或内容任务。
- 页面没有落入通用紫蓝渐变、三卡片 hero、纸色档案默认皮肤等套路。

## 3. 美学一致性

- 全页共享同一套颜色、字体、网格、线条、圆角和动效语言。
- 有变奏，但变奏服务章节差异，不是每章一个系统。
- 纹理和氛围层保持在背景，不压正文和证据。
- 强调色有语义用途，没有大面积滥用。

## 4. 字体与层级

- display / body / mono 角色清楚，字体真实加载。
- 标题字体和正文字体有气质差异。
- 中文正文有合适字体覆盖。
- 字号走比例系统，不是随手数字。
- 第一眼、第二眼、第三眼的阅读路径清楚。

## 5. 结构与网格

- 正文、图表、注释、卡片、流程节点和导航都落在同一套网格。
- 连续正文只有一个 measure。
- 右边缘只落在少数几个明确宽度档，不长长短短。
- 桌面端真实使用横向空间，不是一根窄列悬在中间。
- 移动端无页面级横向滚动。

## 6. 版式与内容塑形

- 每章版面与 plan 的章版面映射一致。
- 除对等模板章外，章节之间有内容驱动的版面差异。
- 长论述不是原样照贴，至少有导语、拉引、小标题、边注或判断提块。
- 没有文字墙、表格墙、卡片墙。
- 编号、标签、眉标和分隔符都承载真实信息。

## 7. 图表与图示

- 原文给数字才画带刻度的图。
- 定性关系没有被画成精确量化。
- 图表使用位置 / 长度优先，避免饼图、环图、3D 和装饰性 chartjunk。
- 节点图同层等宽等高、连接线统一、方向清楚。
- 图表和图示继承页面 token。
- 颜色不是唯一编码，关键值直接标注。

## 8. 动效与交互

- 动效服务主题、层级、流程、图表揭示或章节转场。
- 没有到处 fade-in、hover 花活或自定义光标等零散特效。
- 动效不影响二次阅读。
- `prefers-reduced-motion` 有降级。
- 导航、tab、按钮、链接的焦点态和当前态清楚。

## 9. 可访问性与分享

- 语义标签完整：`header`、`nav`、`main`、`section`、`aside`、`footer` 按需使用。
- 标题层级合理。
- 文本对比度达标。
- 链接文本有意义。
- 表格有清楚语义，必要时有 caption。
- 有 `title`、`description`、`viewport`。
- 单文件 HTML 可直接打开和分享。
- JS 关闭时关键内容仍可读。

## 10. 视觉验证

- 环境支持时，用浏览器截图检查桌面端和移动端。
- 至少检查首屏、一个核心章节、一个图表 / 图示区域和最长表格或最长文本块。
- 截图中不得出现文本重叠、内容被裁切、移动端页面级横向滚动、首屏焦点偏离、图表标签不可读。
- 若无法截图，最终回复中说明未验证视觉截图，并改用静态检查覆盖布局、响应式和可访问性风险。
