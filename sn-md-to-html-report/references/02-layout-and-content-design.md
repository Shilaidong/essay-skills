# 02 · 版式与内容设计

本文件用于 `plan.md` 的「信息结构与版式策略」和「逐章内容设计表」。它在美学方向之后使用：美学方向决定页面的世界，版式与内容设计决定读者如何穿过这个世界。

核心原则：**结构即信息**。布局、编号、标签、分隔符、导航、章节版面、图表和动效都必须传递真实的信息关系。

## 1. 先定阅读层级

为整页和每章分别回答：

- 第一眼看什么：主题代表物、核心判断、主图、关键冲突、任务入口。
- 第二眼看什么：支撑证据、分组、对比、流程、风险。
- 第三眼看什么：口径、来源、限制、附录、细节。

再把层级翻译成设计手段：

- 尺度：大标题、主装置、宽块、正文、注释。
- 重量：字重、密度、线条、底色、留白。
- 位置：左上入口、中心焦点、边注、脚注、附录。
- 节奏：密集段之后给呼吸，重要转折处加停顿。

## 2. 选择页面拓扑

不要默认单页滚动。先判断读者是在“读”还是在“用”。

| 拓扑 | 适合 | 代价 |
|---|---|---|
| 单文档滚动 | 研究、复盘、白皮书、叙事，读者通读一遍 | 非线性查阅较弱 |
| 多视图切换 | 多对象、多赛道、多榜单、对等视角 | 需要 DOM 全量、无 JS 降级、移动端降级 |
| dashboard hub | KPI、指标族、监控、经营简报 | 容易变成数字墙，必须有下钻结构 |
| 向导 / stepper | 决策件、流程、方案评估 | 不适合跳读和并行比较 |

如果选择非滚动拓扑，必须保证所有内容仍在 DOM 中，JS 关闭时顺序可读，移动端不会横向溢出。

## 3. 选择页面骨架

骨架决定横向空间怎么用。

| 骨架 | 适合 | 要点 |
|---|---|---|
| 单栏长读 | 连续论证、叙事、备忘 | 正文走稳定 measure，图表可出栏 |
| 主栏 + 边栏 | 来源、术语、口径、TOC、限制很多 | 边栏服务参照，不当装饰 |
| 宽幅多区网格 | 多对象比较、数据密集、市场地图 | 区内正文仍保持可读行宽 |
| 受限正文 + 全幅特写 | 专题、叙事、核心装置 | 全幅必须被内容挣得 |

所有骨架共享同一套网格和宽度档。正文、图表、卡片、流程节点、边注和导航都要归位，避免各自居中、长长短短。

## 4. 导航与结构元素

导航和编号都不是装饰。

导航选择：

- 短且线性的报告：通常无导航，最多进度提示。
- 长且需要跳读：TOC、scrollspy、锚点、回顶。
- 多视图 / dashboard：tab、segmented、磁贴入口或分页。
- IA 5-7 个并列项：顶栏可行。
- IA 超过 7-9 个或需要持续参照：侧栏更稳。

结构元素使用规则：

- 编号只用于真实序列、阶段、步骤、时间线、排名或条款。
- 标签只用于真实分类、状态、证据类型、风险等级或口径。
- 分隔符用于分组、节奏或状态变化，不用于填空。
- 眉标应传递当前位置、类别或证据属性，不写模板腔。

## 5. 章节版面

每章都要选择版面。构图不是首屏特权。

| 章节版面 | 适合 | 降级 |
|---|---|---|
| 单列流 | 连续论证、必须按序读 | 仍要导语、拉引、提块、小标题 |
| 双栏对照 | 新旧、多空、A/B、政策前后 | 小屏上下交替 |
| 主文 + 边注 | 来源、口径、限制、hedge 密集 | 小屏内联注释 |
| sticky 分屏 | 一张核心图讲完整章 | 小屏图在上、文在下 |
| 不对称网格 | 一个主判断 + 多个支撑件 | 小屏按主次顺序堆叠 |
| 全幅破格 | 核心装置、关键图表、章节转折 | 小屏收回安全边距 |
| 横向画廊 / small multiples | 多个同构对象并比 | 小屏改为可滚动或两列 |

**这张表是词汇表，不是菜单。** 全部从表里选会得到家族相似的页面。要求：

- **至少一个章节版面是为这份内容自创的、表里没有的**，从结构想法推导，在 plan 写明它为什么只属于这份内容。
- 自创版面同样要给小屏降级。

在 `plan.md` 写成：

```text
章节 -> 内容形状 -> 章节版面 -> 呈现形式 -> 排版处理 -> 证据贴附
```

## 6. 内容塑形

Markdown 是素材，不是版面。逐段或逐判断识别信息形状，再决定呈现。

| 信息形状 | 合适呈现 |
|---|---|
| 时间 / 演进 / 步骤 | 时间线、阶段轴、步骤条 |
| 对比 / 取舍 / 多方 | 双栏对照、矩阵、并列表 |
| 层级 / 递进 / 衰减 | 阶梯、树、缩进层级 |
| 因果 / 流程 / 链路 | 流程链、因果链、泳道图 |
| 构成 / 占比 | 100% 条、比例条、分解图 |
| 多点数值 / 可比量级 | 柱、条、折线、斜率、散点、small multiples |
| 定义 / 口径 / 术语 | 定义表、口径表、悬挂缩进 |
| 并列要点 | 索引行、规则线列表 |
| 判断 + 依据 | 判断提块 + 紧邻证据与来源 |
| 长论述 | 导语、拉引、小标题、边注、关键判断提块 |

避免三种墙：

- 文字墙：连续多段纯正文，只加粗不算设计。
- 表格墙：连续多张表，或把所有可比数字都拍成表格。
- 卡片墙：有主次、顺序、因果的内容被做成等权卡片。

## 7. 图表与图示

图表不只是“把数字画出来”，它要同时做到断言守恒、可读、统一、好看。

硬规则：

- 原文给数字，才画带刻度的量化图。
- 原文只有定性关系，只做排序、相对长度或无刻度表达。
- 柱 / 条图基线从 0。
- 用位置和长度优先，不滥用饼图、环图、3D、面积团。
- 颜色不是唯一编码，必须配合位置、形状、标签或线型。
- 图表字号、线宽、间距、颜色继承页面 token。

节点型图示：

- 同层节点等宽等高，走网格。
- 连接线有统一样式，不用默认箭头字符堆。
- 主方向一致，少交叉。
- 配色克制，不每框一个 hue。
- 标签分层：类型弱、主信息强、条件就近。

## 8. 证据贴附

证据应该贴近它支持的判断。

- 关键数字旁标单位、口径、来源。
- 限制、hedge、数据缺口靠近相关判断。
- 反方证据不要藏在文末。
- 附录承接细节，但不能让核心证据离判断太远。
