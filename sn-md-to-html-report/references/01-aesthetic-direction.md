# 01 · 短设计计划

本文件用于 `plan.md` 的「短设计计划」阶段。目标不是挑一个好看的皮肤，而是先 brainstorm 2-3 个能从报告所属领域、读者任务和内容气质里推导出来的设计方向，再收敛为一套紧凑的视觉基因：Color、Type、Layout 和 Signature。

先完成本文件，再进入 `02-layout-and-content-design.md`。不要一边写 HTML 一边找风格。

## 1. 提取内容信号

只记录能帮助确定视觉世界的信号：

- **领域**：金融、教育、城市、医疗、AI、制造、政策、消费、游戏、能源等。
- **场景与工具**：账本、控制台、地铁图、课表、病历、试验台、工单、地图、航线、仪表盘。
- **物理质感**：纸张、玻璃、金属、网格、扫描线、墨迹、地形、档案、终端、标牌。
- **内容任务**：解释、比较、决策、复盘、预警、说服、检索、展示。
- **情绪温度**：冷静、紧迫、精密、开放、沉稳、实验性、公共性、纪实感。

这些信号不等于页面元素清单。它们用于推导风格，不用于机械堆装饰。

## 2. 选择首屏代表物

首屏从主题领域中最具代表性的事物或机制开篇。它可以是标题、图片、动画、实时演示、互动结构或视觉隐喻。

选择标准：

- 它属于这个领域，一眼能建立“这是什么世界”。
- 它能承载报告主题，而不是只负责好看。
- 它不是通用模板：大数字 + 小标签 + 统计卡 + 渐变强调只在数字本身就是最自然入口时使用。
- 它不会制造事实断言。视觉隐喻可以来自领域，页面里的数字、案例、来源和结论强度必须来自原报告。

## 3. Brainstorm 2-3 个短设计方向

在 `plan.md` 中先列候选，再选择。候选必须明显不同，至少一个明暗相反；每个候选都必须说明它与内容信号的联系，不能只是风格名或视觉情绪。

每个候选用 compact token system 表达：

```text
方向名：
  内容联系：来自哪些领域、工具、材料、证据关系或读者任务
  Color：4-6 个命名 hex 值，每个说明语义用途
  Type：display / body / utility 字体角色
  Layout：一句话概念 + ASCII wireframe
  Signature：这页唯一会被记住的设计元素
```

选择时写明：

- 为什么选中方向最贴合内容任务。
- 为什么落选方向不如它。
- 把这套方向换到完全不同主题的报告上是否还成立；若成立，说明方向太通用，必须改到能体现当前内容。

## 4. 配色

配色先由主题决定色相，再由规则保证协调与可读性。

- **base hue**：从领域和情绪温度来，不从个人偏好来。
- **中性色**：不要纯灰；背景、正文、边线用同一 base hue 的低 chroma 版本，形成同一个色温。
- **强调色**：一个主强调色，最多一到两个辅助色。每个非中性色必须有语义用途，如风险、机会、引用、当前状态。
- **禁忌**：无语义的彩虹色、紫蓝渐变默认 hero、为“科技感”随手加高饱和蓝紫。

## 5. 字体

字体是页面性格的一部分，不只是内容载体。

- **display**：负责标题、首屏焦点和强记忆点，要有明确气质。
- **body**：负责长文阅读，要稳、清楚、耐读。
- **mono / numeric**：负责数字、代码、时间戳、表格和坐标感。
- display 与 body 要形成角色差异：一个有性格，一个克制耐读。不要两个都抢戏，也不要两个都平。
- 不要使用其他项目常用的字体系列作为默认答案。Inter / Roboto / Arial / system stack 不应成为主角。
- 中文正文必须有合适的中文字体兜底，如 Noto Sans SC / Noto Serif SC / 思源黑体 / 思源宋体。
- 数字密集处使用 `font-variant-numeric: tabular-nums`。

## 6. 纹理、材质与氛围

纹理和氛围是建立领域感的高杠杆手段，但必须退到内容之后。

原则：

- 纹理必须来自主题领域，通过换报告测试。
- 低对比、低频率、低存在感，保证图地关系。
- 优先用 CSS / SVG 手写，保持单文件轻量。

## 7. 动效与氛围

动效服务主题、阅读或信息理解，不是散落的特效。

护栏：

- 不反复骚扰读者；入场动画通常只发生一次。
- 不让动效压过文字和证据。
- 必须提供 `prefers-reduced-motion` 降级。

## 8. 不可遗忘点

每个报告页面都需要一个能被截图记住的结构。它可能是：

- 主题领域代表物的首屏构图。
- 一个独特的图表或流程装置。
- 一种贯穿全页的版面机制。
- 一个能表达报告核心关系的视觉隐喻。

检查问题：

- 任意关键屏被截图发出去，别人能凭哪个点记住这是这份报告？
- 这个点来自主题领域和内容任务，还是通用装饰？
- 它在首屏或核心章节真实落地了吗？
